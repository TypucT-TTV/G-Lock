# Списки: Вайтлист и Блэклист

## Где хранятся

Оба списка (и выбранный язык интерфейса) хранятся в одном файле —
**`data.json`**, рядом с исполняемым файлом G-Lock. Формат — простой JSON:

```json
{
    "whitelist": {
        "185.56.64.10": "Друг1",
        "185.56.64.11": "Друг2"
    },
    "blacklist": {},
    "language": "ru"
}
```

Ключ — IP-адрес, значение — произвольное имя-подпись для удобства (само имя
никак не участвует в фильтрации пакетов, используется только для отображения
в интерфейсе). Файл читается при старте (`ConfigData.__init__` — если файла
нет, создаётся с пустыми списками и языком по умолчанию `ru`) и
перезаписывается целиком при каждом сохранении.

Оба списка реализованы как **синглтоны** (`Whitelist`/`Blacklist`,
`metaclass=Singleton`) — то есть в рамках одного запуска программы
существует ровно один объект каждого списка, и все части интерфейса (кнопки
запуска сессий, редактор списков) всегда видят одни и те же, актуальные
данные без необходимости их синхронизировать между собой вручную.

## Открытие редактора

Кнопка **"Списки (вайт/блэклист)"** открывает отдельное окно с двумя
вкладками — Вайтлист и Блэклист. У каждой вкладки — таблица (имя + IP) и три
кнопки: Добавить, Изменить, Удалить.

## Добавление записи

1. Открывается форма с полями "Имя" и "IP-адрес".
2. При нажатии "Сохранить" выполняется валидация **в таком порядке**:
   - Имя проверяется на дубликат (`NameValidator.validate_get`) — просто
     `if global_list.has(name): ошибка`. Пустое имя дубликатом не считается
     (кроме случая, если пустое имя уже где-то использовано).
   - IP проверяется на формат (`IPValidator.validate_get`) — сначала
     регулярным выражением на общий вид `ЧЧЧ.ЧЧЧ.ЧЧЧ.ЧЧЧ` (числа 0–255),
     затем дополнительно через `socket.inet_aton` (более строгая проверка,
     отсекает то, что регулярка могла пропустить).
   - Отдельно проверяется, что такого IP ещё нет в списке.
3. Если всё прошло — запись добавляется (`global_list.add(ip, name)`) и
   сразу сохраняется на диск (`global_list.save()`), таблица обновляется.
4. Если что-то не прошло — под полями показывается конкретная причина
   ("Имя уже в списке" / "Некорректный IP" / "IP уже в списке"), форма не
   закрывается, можно поправить и попробовать снова.

## Редактирование записи

Форма открывается с уже заполненными текущими значениями. Особенность:
**проверка имени на дубликат пропускается, если имя не изменилось** — иначе
редактирование записи без смены имени всегда бы упиралось в "имя уже в
списке" (ведь оно там уже есть — это же та самая запись). IP при этом всегда
проверяется на формат и на дубликат (кроме случая, когда IP тоже не менялся).

Технически редактирование — это не "изменение на месте", а **удаление
старой записи и добавление новой** с введёнными значениями (после успешной
валидации). Если между удалением и добавлением произойдёт сбой — теоретически
можно потерять запись, но поскольку обе операции идут синхронно одна за
другой без операций ввода-вывода между ними, риск на практике отсутствует.

## Удаление записи

Просто выбрать строку в таблице и нажать "Удалить" — без дополнительного
подтверждения. Удаление сразу сохраняется на диск.

## Как список используется при запуске сессии

При запуске "Сессия по вайтлисту" (`Menu.launch_whitelisted_session`) или
"Сессия по блэклисту" (`Menu.launch_blacklisted_session`) список **читается
заново** и **повторно валидируется** (`IPValidator.validate_get` на каждую
запись) — если какая-то запись в файле окажется некорректной (например,
файл `data.json` был отредактирован вручную и там оказался мусор), эта
конкретная запись будет пропущена с сообщением в консоль/лог, но **не
остановит** запуск сессии с остальными, валидными адресами.

## Ограничения текущей реализации

- Нет отдельного подтверждения при удалении — удаление необратимо мгновенно
  (единственный способ вернуть — добавить IP заново вручную).
- Имя используется только для удобства человека — не проверяется на
  осмысленность, можно оставить пустым.
- Нет проверки, что добавляемый IP не относится к диапазонам Rockstar/Azure
  (см. `TODO` в исходном коде `validator/ip.py`) — это осознанно оставлено
  как есть в текущей версии.
