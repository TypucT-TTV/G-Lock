# Интерфейс: разбор каждого элемента

Главное окно (`gui/main_window.py`) целиком построено на `tkinter`, без
сторонних UI-библиотек. Ниже — что происходит при взаимодействии с каждым
конкретным элементом, сверху вниз.

## Заголовок: цветной индикатор + текстовый статус

Кружок и подпись рядом с ним (например, "🟢 Открыто") отражают состояние
`Context` в реальном времени:

- **Зелёный / "Открыто"** — ни один фильтр не активен, WinDivert-перехват
  выключен, все подключаются как обычно.
- **Красный / "ЗАПЕРТО"** — активна Locked-сессия (`Context.is_locked() ==
  True`).
- **"Активно: <тип>"** — активен любой другой фильтр (Solo, Whitelist,
  Blacklist и т.д.), кроме Locked.

Этот же текст дублируется в заголовке окна (`G-Lock 1.0.0 - Открыто`), чтобы
статус был виден даже когда окно свёрнуто в панель задач и видно только его
название во всплывающей подсказке/Alt+Tab.

Обновление происходит не мгновенно "изнутри" клика — оно завязано на
`Context.on_change`: любое изменение состояния фильтра (через кнопку **или**
через F9, из любого потока) кладёт задание на обновление в очередь
(`main_queue`), которую сам интерфейс проверяет каждые 100 мс
(`_pump_queue`). Это сделано специально, чтобы код кнопок и код горячей
клавиши F9 не должны были напрямую трогать виджеты Tk из чужого потока —
tkinter категорически не любит, когда с его виджетами работают не из
главного потока.

## Кнопка "Остановить сессию"

Отдельная кнопка, специально размещённая **не** среди кнопок запуска, а
отдельно, сразу под статусом. Причина — чтобы выключение любого активного
режима было явным, предсказуемым действием с одной, всегда одной и той же
кнопкой, а не "нажми ту же кнопку ещё раз и надейся, что она правильно
переключится".

- **Активна** (можно нажать), только если `Context.is_filter_running() ==
  True` — то есть реально что-то запущено. Иначе кнопка визуально приглушена
  и клики по ней ничего не делают.
- При нажатии вызывает `Menu.stop_session()` в отдельном фоновом потоке
  (см. ниже, почему в фоновом), которая просто делает
  `Context.kill_latest_filter()` — останавливает текущий фильтр и
  возвращает статус к "Открыто".
- Работает одинаково для **любого** активного режима — не только для Lock,
  но и для Solo/Whitelist/Blacklist и т.д.

## Кнопки типов сессий (с индикатором)

Каждая кнопка сессии (Solo-сессия, Сессия по вайтлисту, Сессия по блэклисту,
Авто-вайтлист сессии, Закрытая сессия, Пустая сессия) содержит **встроенный
кружок-индикатор** слева от текста, нарисованный прямо на той же канве, что и
сама кнопка (не отдельным виджетом рядом — так проще добиться одинаковых
отступов слева/справа у всех кнопок).

- **Кружок горит зелёным**, если именно этот тип фильтра сейчас активен.
- **Серый** — не активен.

Важный нюанс: `Context` знает только "какой ИМЕННО класс фильтра сейчас
работает" — а не "какая кнопка меню его запустила". Поэтому:
- Индикаторы "Solo-сессия" и "Пустая сессия (стать хостом)" загораются
  **вместе**, потому что обе кнопки технически запускают один и тот же
  `SoloSession` — просто с разным описанием для разных сценариев
  использования ("хочу поиграть один" vs. "хочу гарантированно стать
  хостом").
- То же самое для "Сессия по вайтлисту" и "Авто-вайтлист сессии" — обе дают
  `WhitelistSession`.

Это не баг, а точное отражение реальности: под капотом это действительно один
и тот же тип защиты.

При нажатии на кнопку сессии: сначала показывается диалог подтверждения с
описанием, что делает этот режим (см. `02_Типы_сессий.md` за текстом каждого
описания), и только при согласии — реальный запуск.

Кнопки без индикатора (**Кикнуть неизвестных**, **Кикнуть по IP**, **Списки**,
**Поблагодарить ТурисТа**, **Выход**) не участвуют в этой системе — они либо
временные разовые действия (кик автоматически отключается сам через 10
секунд), либо вообще не про сетевые фильтры.

## Почему кнопки не блокируют окно

Каждая кнопка (кроме "Списки" и "Выход", это чисто локальные UI-действия)
при нажатии запускает соответствующую функцию `Menu.*` в **отдельном фоновом
потоке** (`threading.Thread(..., daemon=True).start()`), а не напрямую в
потоке интерфейса. Причина: некоторые действия занимают время (60 секунд сбор
IP при авто-вайтлисте, диалоги подтверждения, которые должны дождаться
ответа) — если бы это выполнялось прямо в главном потоке Tk, окно бы
"зависало" на всё это время, не реагируя даже на перемещение/закрытие.

Диалоги подтверждения (Да/Нет), мультивыбора (список IP с чекбоксами) и
прогресс-бары с отменой на самом деле создаются на главном потоке (иначе Tk
работать не будет), но **вызывающий их код бизнес-логики** блокируется на
фоновом потоке, ожидая ответа — обмен идёт через потокобезопасную очередь
(`gui/adapter.py`). Подробностей этого механизма в интерфейсе не видно —
снаружи это выглядит как обычный модальный диалог.

## Переключатель языка (RU / EN)

Две кнопки внизу окна. Текущий выбранный язык показывается как неактивная
(приглушённая) кнопка — потому что переключаться "на тот же язык, что уже
выбран" бессмысленно.

При нажатии на другой язык:
1. `i18n.set_language(...)` сохраняет выбор в `data.json` (тот же файл, где
   лежат вайтлист и блэклист) — сохраняется между перезапусками.
2. Всё окно полностью **пересоздаётся** (`_rebuild()` — все виджеты
   уничтожаются и строятся заново) с новыми текстами. Активные сетевые
   фильтры при этом не трогаются — пересоздаётся только интерфейс, а не
   `Context`/фильтры.

## Иконка в панели задач

Меняется синхронно со статусом (зелёный круг = открыто, красный = заперто).
Особенность реализации: иконки **не** хранятся как файлы в проекте — они
генерируются на лету из чистого Python (`gui/icons.py`), без Pillow и без
внешних библиотек: свой мини-энкодер PNG (через `struct`+`zlib`) рисует
закрашенный круг нужного цвета, оборачивает его в формат `.ico` и сохраняет
во временный файл. Оба варианта (зелёный/красный) генерируются один раз за
всё время работы программы и переиспользуются.

Иконка применяется двумя способами одновременно — `root.iconbitmap(...)`
(стандартный способ Tk) и напрямую через Windows API `SendMessage(...,
WM_SETICON, ...)` — второй способ подстраховка на случай, если первый не
обновит именно значок в панели задач на некоторых версиях Windows/Tk (только
заголовок окна).

## Общий вид окна

Кнопки и текст расположены поверх тёмного фона с приглушёнными геометрическими
фигурами (шестиугольники, тонкие диагональные линии, пара колец) — это чисто
декоративный слой, никакой функциональной роли не играет, дорисовывается один
раз при открытии окна отдельным `Canvas`, лежащим "под" остальными
элементами.
