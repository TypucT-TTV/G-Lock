# Списки: Вайтлист и Блэклист

## Где хранятся

Оба списка (и выбранный язык интерфейса) хранятся в одном файле —
**`data.json`**, рядом с исполняемым файлом G-Lock. Формат — простой JSON:

```json
{
    "whitelist": {
        "185.56.64.10": "",
        "185.56.64.11": ""
    },
    "blacklist": {},
    "language": "ru"
}
```

Ключ — конкретный IP-адрес или **CIDR-диапазон подсети** (например, `52.139.169.0/24`), значение — пустая строка `""` (поскольку имена были полностью убраны из списков для упрощения добавления «в один клик»). 

Файл читается и записывается строго в кодировке **`utf-8`**. Чтение происходит при старте (`ConfigData.__init__`), а перезапись — при изменении списков через интерфейс или интерактивный лог.

Оба списка реализованы как **синглтоны** (`Whitelist`/`Blacklist`,
`metaclass=Singleton`) — то есть в рамках одного запуска программы
существует ровно один объект каждого списка, и все части интерфейса (кнопки
запуска сессий, редактор списков, лог активности) всегда видят одни и те же, актуальные
данные.

## Открытие редактора

Кнопка **"Списки (вайт/блэклист)"** открывает отдельное окно с двумя
вкладками — Вайтлист и Блэклист. У каждой вкладки отображается таблица, содержащая только IP-адреса, и четыре кнопки управления:
1. **Добавить** — добавление нового адреса.
2. **Изменить** — редактирование выбранного адреса.
3. **Удалить** — мгновенное удаление выбранного адреса из списка.
4. **Очистить** — полное удаление всех записей из текущего списка (очень удобно, так как у большинства игроков динамические IP, меняющиеся к следующему стриму).

## Добавление записи

1. Открывается форма с единственным полем "IP-адрес".
2. При нажатии "Сохранить" выполняется валидация:
   - IP проверяется на формат (`IPValidator.validate_get`) — сначала
     регулярным выражением на общий вид `ЧЧЧ.ЧЧЧ.ЧЧЧ.ЧЧЧ` (числа 0–255),
     затем дополнительно через `socket.inet_aton` (отсекает то, что регулярка могла пропустить).
   - Для белого списка выполняется проверка на Rockstar-релеи (`dynamic_blacklist`) — попытка добавить релей Rockstar блокируется с предупреждением, чтобы не открыть лазейку для туннель-атак.
   - Проверяется, что такого IP ещё нет в списке.
3. Если всё прошло — запись добавляется (`global_list.add(ip, "")`) и
   сразу сохраняется на диск (`global_list.save()`), таблица обновляется, а активная сессия перезапускается в фоне.
4. Если что-то не прошло — под полем показывается причина ("Некорректный IP" / "Такой IP уже есть в списке"), форма не закрывается.

## Редактирование записи

Форма открывается с уже заполненным текущим значением IP. Редактирование — это технически удаление старой записи и добавление новой с введенным значением после успешной валидации. Изменения сохраняются на диск и вызывают автоматическую перезагрузку активной сессии.

## Удаление и очистка списка

При нажатии кнопок **"Удалить"** или **"Очистить"** изменения немедленно сохраняются на диск, а активный фоновый фильтр сессии автоматически перезапускается «на лету» в фоновом потоке, применяя новые правила защиты мгновенно.

## Добавление записей из интерактивного лога

Вы можете добавлять IP-адреса в белый или черный список напрямую из панели сетевой активности в главном окне без ввода имен и без всплывающих окон:
- Нажатием кнопки `🟢` перед строкой IP — мгновенно в белый список.
- Нажатием кнопки `🔴` перед строкой IP — мгновенно в черный список.
- Через меню правой кнопки мыши (ПКМ) — опции также срабатывают мгновенно без диалогов.
При этом защита также обновляется «на лету».
