# Прочее: донат, выход, язык, сборка

## Кнопка "Поблагодарить ТурисТа"

Простейшая функция (`Menu.open_donation`): открывает ссылку
`https://www.donationalerts.com/r/typuct_donate` в браузере по умолчанию
(`webbrowser.open`). Никакого сетевого запроса из самого G-Lock не делается
— браузер открывается как отдельный процесс, дальше всё происходит уже в нём.

## Кнопка "Выход"

`root.destroy()` — закрывает главное окно и завершает процесс. Важный нюанс:
**активная сессия при этом не останавливается автоматически явным образом** —
процесс-фильтр (если запущен) не получает отдельной команды "остановись",
а завершается вместе со всем процессом G-Lock, так как все
процессы-фильтры создаются с `daemon=True` (демон-процессы автоматически
завершаются, когда завершается родительский процесс). На практике разницы
для пользователя нет: закрытие G-Lock в любом случае прекращает всю
фильтрацию трафика.

То же самое происходит при закрытии окна крестиком — оба пути (кнопка
"Выход" и крестик в заголовке окна) ведут в один и тот же обработчик
`_on_quit`.

## Переключатель языка

Подробно описан в `01_Интерфейс.md`. Коротко: выбор языка хранится в
`data.json` (том же файле, где вайтлист/блэклист), переключение полностью
пересоздаёт интерфейс с новыми текстами, не трогая активные сетевые фильтры.

Все переводимые строки живут в одном месте — `gui/i18n.py`, как плоский
словарь `{язык: {ключ: текст}}`. У каждого ключа есть форма для русского и
для английского; если для какого-то ключа перевод отсутствует на выбранном
языке — это баг в `i18n.py`, а не ожидаемое поведение (в текущей версии оба
языка заполнены полностью).

## Сборка исполняемого файла

G-Lock собирается в один `.exe`-файл через `PyInstaller`
(`g_lock/build.py`, запускается командой `poetry run build`). Из
интересного:

- Флаг `--uac-admin` встроен в манифест самого `.exe` — двойной клик сам
  вызовет запрос UAC на права администратора, никаких дополнительных
  ярлыков/скриптов для этого не требуется.
- Иконка приложения и метаданные версии (`ProductName`, `FileVersion` и
  т.д.) берутся из `spec/logo.ico` и `spec/version.txt`.
- Собранный `.exe` **не включает** в себя `db.json` (резервный список
  Rockstar/Azure диапазонов, см. `06_Обработка_ошибок.md`) — это отдельный
  файл, который нужно класть **рядом** с `.exe` вручную при распространении
  сборки. Без него, если ещё и сеть недоступна при первом запуске на новом
  компьютере, программа не сможет запуститься (см. соответствующий раздел в
  `06_Обработка_ошибок.md`).
- Файлы `data.json`, `history.log`, `debugger.log` и папка `logs/` создаются
  автоматически при первом запуске в той же папке, где лежит `.exe` — их не
  нужно (и не нужно пытаться) создавать заранее.

## Структура документации

- `00_Обзор_архитектуры.md` — WinDivert, модель процессов и приоритетов,
  эвристика по размеру пакета.
- `01_Интерфейс.md` — разбор каждого элемента главного окна.
- `02_Типы_сессий.md` — подробно про каждый режим сессии.
- `03_Горячая_клавиша_F9.md` — как устроено переключение Lock через F9.
- `04_Списки_вайтлист_блэклист.md` — редактор списков, валидация, формат.
- `05_Лог_подключений.md` — фоновый лог "кто и когда подключался".
- `06_Обработка_ошибок.md` — запуск, сбои, таймауты сетевых запросов.
- `07_Прочее.md` — этот файл: донат, выход, язык, сборка.

Отдельно, для новых пользователей — краткие практические инструкции
(без технических подробностей) лежат в этой же папке:
`Инструкция_RU.txt` и `Instructions_EN.txt`.
