# Руководство разработчика G-Lock

Добро пожаловать в руководство разработчика G-Lock! Этот документ содержит информацию о внутреннем устройстве кодовой базы, архитектуре и процессах разработки, чтобы облегчить погружение в проект.

---

## 1. Быстрый старт (Настройка окружения)

Для разработки и запуска G-Lock вам понадобятся:
- **ОС**: Windows 10/11 x64 (драйвер WinDivert поддерживает только Windows).
- **Python**: Версия `>=3.10` и `<3.12`.
- **Права Администратора**: Необходимы для регистрации драйвера WinDivert и захвата сетевых пакетов.

### Шаги установки:
1. Клонируйте репозиторий.
2. Установите менеджер зависимостей `poetry` (если он еще не установлен):
   ```bash
   pip install poetry
   ```
3. Установите зависимости проекта из корневой папки:
   ```bash
   poetry install
   ```
4. Настройте pre-commit хуки для автоматической проверки стиля кода:
   ```bash
   poetry run pre-commit install
   ```

### Запуск в режиме разработки:
Запустите консоль с правами администратора и выполните:
```bash
poetry run python g_lock
```

## 2. Архитектура проекта

Кодовая база G-Lock разделена на несколько ключевых модулей в папке `g_lock/`:

```
g_lock/
├── __main__.py          # Точка входа, инициализация логов, проверка прав администратора
├── build.py             # Скрипт автоматизации сборки PyInstaller
├── config/              # Чтение, сохранение и валидация конфигурационных файлов
│   ├── configdata.py    # Базовые классы конфигурации
│   └── globallist.py    # Работа со списками IP (Whitelist, Blacklist)
├── dispatcher/          # Управление активными сетевыми фильтрами
│   └── context.py       # Контекст сессии (отслеживание активного процесса фильтрации)
├── gui/                 # Пользовательский интерфейс (Tkinter)
│   ├── main_window.py   # Главное окно с кнопками управления сессией
│   ├── widgets.py       # Кастомные виджеты кнопок и полей ввода
│   └── dpi.py           # Инициализация поддержки масштабирования высокого DPI
├── menu/                # Описание структуры главного меню и его команд
├── network/             # Логика работы с сетью и перехвата пакетов
│   ├── sessioninfo.py   # Выявление Rockstar Tunnels и сбор информации
│   └── sessions.py      # Классы фоновых процессов фильтрации для каждого режима
└── validator/           # Вспомогательные валидаторы для IP-адресов и имен
```

### Главные архитектурные концепции:

#### 1. Многопроцессность (Multiprocessing)
Для предотвращения блокировки графического интерфейса Tkinter (`main_window.py`) логика захвата пакетов WinDivert вынесена в отдельные процессы.
- Каждый тип сессии (например, `SoloSession`, `WhitelistSession`) наследуется от `AbstractPacketFilter` и запускается как отдельный `multiprocessing.Process` (см. [sessions.py](file:///g:/g_lock-3.5.0/g_lock/network/sessions.py)).
- Менеджер контекста `Context` ([context.py](file:///g:/g_lock-3.5.0/g_lock/dispatcher/context.py)) гарантирует, что в любой момент времени запущен ровно один процесс фильтрации. При включении нового режима старый процесс принудительно терминируется с помощью `.terminate()`.

#### 2. Отрисовка GUI на Canvas (Glassmorphism)
В отличие от стандартных Tkinter приложений, главное окно G-Lock использует кастомные Canvas-элементы для имитации прозрачности («стеклянный» стиль).
- Все кнопки главного меню рисуются непосредственно на `tk.Canvas` методом `create_polygon` с использованием штриховки `stipple="gray50"` (см. [widgets.py](file:///g:/g_lock-3.5.0/g_lock/gui/widgets.py)).
- Любые изменения размеров или координат виджетов должны учитывать DPI-масштаб, полученный через `dpi.get_scale_factor()` из [dpi.py](file:///g:/g_lock-3.5.0/g_lock/gui/dpi.py).

#### 3. Сетевой лог (ConnectionLogger)
Фоновый процесс `ConnectionLogger` ([connectionlog.py](file:///g:/g_lock-3.5.0/g_lock/network/connectionlog.py)) работает на протяжении всего времени жизни приложения в режиме пассивного прослушивания (`SNIFF`).
Чтобы избежать конфликтов блокировок файлов при многопроцессности, сам дочерний процесс логгера отправляет сырые события пакетов через потокобезопасную мультипроцессорную очередь `_log_queue` в главный процесс. Запись в ежедневные лог-файлы на диске и обновление интерактивной таблицы логов на экране выполняются на выделенном фоновом потоке внутри главного GUI-процесса.

## 3. Сборка проекта

Для сборки автономного исполняемого файла `.exe` используется библиотека `PyInstaller`, упакованная в скрипт Poetry:

```bash
poetry run build
```

Скрипт сборки [build.py](file:///g:/g_lock-3.5.0/g_lock/build.py) считывает метаданные проекта, настраивает пути к ресурсам (`db.json` и `data.json`) и запускает PyInstaller с флагами `--onefile` (один файл) и `--noconsole` (скрыть окно консоли при запуске). Результат компиляции сохраняется в папку `dist/`.
