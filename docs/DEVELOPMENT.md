# Руководство разработчика G-Lock v2

Добро пожаловать в руководство разработчика нового G-Lock (версии 2.x)! Приложение было полностью переписано с Python/Tkinter на современный и легкий стек: **Tauri v2 (Rust)** в качестве бэкенда и **Svelte 5 (TypeScript / CSS)** в качестве фронтенда.

---

## 1. Стек технологий

* **Backend**: Rust (совместимость с `edition = "2021"`, компилятор `rustc 1.77+`).
  * Для захвата и фильтрации трафика используется низкоуровневая библиотека WinDivert (через Rust-биндинги `windivert`).
    * **Внимание**: Биндинги Rust (`windivert-sys` версии `0.10.0`) жестко завязаны на структуру адреса `WINDIVERT_ADDRESS` **WinDivert версии 2.2.2** (размер 80 байт с выравниванием). Использование старых версий драйверов (1.3 / 1.4.3) приведет к повреждению памяти при вызовах FFI и полной блокировке сетевого трафика. Проект должен поставляться строго с бинарными файлами драйвера **WinDivert 2.2.2-A** (`WinDivert.dll`, `WinDivert64.dll`, `WinDivert64.sys`).
  * Для проигрывания звуков — библиотека `rodio`.
  * Для многопоточности и управления фильтрацией — стандартные Rust-каналы (`std::sync::mpsc`) и потоки (`std::thread`).
* **Frontend**: Svelte 5 (TypeScript) + Сборщик Vite + Ванильный CSS.
  * Весь интерфейс упакован в Single-Page App (SPA) с помощью адаптера `@sveltejs/adapter-static`.
  * Реализован красивый темный дизайн с эффектом стеклянного размытия (glassmorphism) и анимациями.
* **Связующий слой**: Tauri v2 IPC.
  * Все команды с фронтенда вызывают Rust-функции (команды) через `invoke()`.
  * События сети (логи подключений, изменение статуса) отправляются в реальном времени с бэкенда на фронтенд через механизм Tauri Events (`emit()`).

## 2. Быстрый старт (Настройка окружения)

Для разработки и запуска G-Lock вам понадобятся:
- **ОС**: Windows 10/11 x64.
- **Node.js**: Версия `>=18.x` (рекомендуется LTS).
- **Rust**: Установленный через [rustup](https://rustup.rs/) (Windows MSVC).
- **Права Администратора**: Необходимы для запуска в режиме разработки/тестирования (поскольку драйвер WinDivert требует админ-привилегий для захвата пакетов).

### Шаги установки:
1. Клонируйте репозиторий.
2. Установите npm-зависимости в корневой директории:
   ```bash
   npm install
   ```

### Запуск в режиме разработки:
Запустите командную строку / терминал от имени администратора и выполните:
```bash
npm run tauri dev
```
Tauri автоматически запустит dev-сервер Vite для фронтенда и скомпилирует Rust-бэкенд с горячей перезагрузкой.

### Сборка проекта:
Для компиляции финального оптимизированного релиза и создания установщиков (MSI, NSIS):
```bash
npm run tauri build
```
Скомпилированный `.exe`-файл будет находиться в `src-tauri/target/release/g-lock-tauri.exe`.  
Готовые пакеты установщиков сохраняются в `src-tauri/target/release/bundle/`.

## 3. Структура проекта

Кодовая база разделена на две основные части:

```
g-lock/
├── src/                      # Фронтенд (Svelte 5 / TypeScript)
│   ├── routes/
│   │   ├── +layout.ts        # Настройка SPA маршрутизации
│   │   └── +page.svelte      # Главный экран интерфейса, дизайн и JS логика
│   └── app.html              # HTML-шаблон страницы
├── src-tauri/                # Бэкенд (Tauri / Rust)
│   ├── Cargo.toml            # Описание зависимостей и версии на Rust
│   ├── tauri.conf.json       # Конфигурация Tauri (окна, иконки, разрешения)
│   ├── icons/                # Ресурсы иконок приложения
│   └── src/
│       ├── main.rs           # Точка входа в приложение
│       ├── lib.rs            # Основной модуль: инициализация Tauri, хуки событий, IPC команды
│       ├── firewall.rs       # Логика сетевой фильтрации через драйвер WinDivert (IPS, режимы)
│       ├── config.rs         # Чтение, сохранение и валидация конфигурации (data.json)
│       └── sound.rs          # Модуль генерации аудио-сигналов (бипперов) без внешних медиа-файлов
```

## 4. Архитектурные требования и правила кода

При внесении изменений обязательно соблюдайте следующие правила:

1. **Безопасность Rockstar-серверов**:
   * Ни при каких условиях не блокируйте служебные пакеты синхронизации (Heartbeats) с размерами payload `12`, `18` и `63` байта.
   * IP-адреса Rockstar/Azure релей-серверов (`Menu.dynamic_blacklist`) должны быть полностью исключены из системы блокировок IPS (Rate Limiter), чтобы избежать ложных банов серверов авторизации.
2. **Многопоточность в Rust**:
   * Захват сетевых пакетов WinDivert является блокирующей операцией (`WinDivert.recv()`). Данная логика должна выполняться в отдельном Rust-потоке (`std::thread::spawn`), чтобы не блокировать графический поток интерфейса.
3. **Версионирование**:
   * Версия проекта описывается в трех файлах: `src-tauri/Cargo.toml`, `package.json` и `src-tauri/tauri.conf.json`. При каждом выпуске патча или функции обновляйте версию в соответствии с SemVer (например, `2.0.0` -> `2.0.1`).
4. **Управление драйвером и блокировка файлов**:
   * Драйвер WinDivert загружается в систему как служба ядра (`WinDivert`). Если процесс компиляции (`npm run tauri build` / `dev`) завершается ошибкой «файл занят другим процессом (os error 32)», это означает, что служба всё ещё активна. 
   * Для ручной остановки используйте команду в консоли от имени Администратора: `sc.exe stop WinDivert` (в PowerShell обязательно указывать `.exe`, так как `sc` является алиасом для `Set-Content`).
