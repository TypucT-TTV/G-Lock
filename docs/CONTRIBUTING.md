# Руководство по контрибуции в G-Lock

Мы рады вашему участию в улучшении G-Lock! Чтобы процесс внесения изменений прошел максимально гладко и быстро, пожалуйста, ознакомьтесь со стандартами разработки, принятыми в этом репозитории.

---

## 1. Процесс внесения изменений

1. **Создайте форк репозитория** и склонируйте его на локальный компьютер.
2. **Создайте тематическую ветку** от ветки `main`:
   ```bash
   git checkout -b feature/my-new-feature
   ```
3. **Внесите изменения в код** и напишите тесты (если применимо).
4. **Убедитесь в прохождении всех локальных тестов и проверок** (см. раздел "Стандарты кода").
5. **Закоммитьте изменения** и отправьте ветку в ваш форк:
   ```bash
   git push origin feature/my-new-feature
   ```
6. **Создайте Pull Request (PR)** в ветку `main` оригинального репозитория.

---

## 2. Стандарты кода

Качество и стиль кодовой базы проекта поддерживаются автоматически при помощи статических анализаторов. Перед созданием Pull Request обязательно запустите следующие проверки:

### 1. Форматирование кода (Black и Isort)
Весь Python-код должен быть отформатирован в соответствии со стандартами Black. Импорты должны быть отсортированы по правилам Isort.
```bash
poetry run black g_lock tests
poetry run isort g_lock tests
```

### 2. Проверка стилей (Flake8)
Убедитесь, что код не содержит синтаксических предупреждений или неиспользуемого импорта:
```bash
poetry run pflake8 g_lock tests
```

### 3. Строгая типизация (Mypy)
Проект настроен на строгую проверку статических типов. Все новые функции и классы должны содержать полные аннотации типов.
```bash
poetry run mypy g_lock
```
Все аннотации типов должны успешно проходить проверку без ошибок. При необходимости используйте встроенные типы или типы из стандартной библиотеки `typing`.

---

## 3. Коммиты

- Пишите понятные и лаконичные сообщения к коммитам.
- Рекомендуется использовать формат коммитов Conventional Commits:
  - `feat: добавить новый режим блокировки по диапазону IP`
  - `fix: исправить баг масштабирования шрифта при DPI > 100%`
  - `docs: обновить руководство разработчика`
  - `test: добавить тесты для валидатора IP-адресов`

---

Создавая Pull Request, вы соглашаетесь с тем, что ваш код будет лицензирован по той же лицензии, что и весь проект G-Lock (LGPL-3.0).
