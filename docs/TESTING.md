# Тестирование в проекте G-Lock v2

В новом проекте G-Lock на базе Tauri тестирование разделено на проверку Rust-бэкенда и тестирование логики работы с драйвером.

---

## 1. Запуск тестов бэкенда (Rust)

Все модульные тесты пишутся на языке Rust и запускаются с помощью стандартного менеджера пакетов `cargo`:

```bash
# Запуск всех тестов в проекте
cargo test

# Запуск тестов с выводом логов stdout
cargo test -- --nocapture
```

---

## 2. Структура тестов

Модульные тесты располагаются внутри самих файлов исходного кода в блоках `mod tests`:
- **Тесты конфигурации**: Тестируют корректность сериализации и десериализации файла настроек `data.json`, проверку путей и значений по умолчанию в `config.rs`.
- **Тесты парсинга пакетов**: Проверяют парсеры пакетов (например, детектирование размеров payload `12`, `18`, `63` для Heartbeats) в `firewall.rs`.

---

## 3. Мокирование и особенности тестирования WinDivert

Поскольку драйвер WinDivert требует прав администратора и работает только на Windows, интеграционные тесты с захватом пакетов не могут быть запущены в не-привилегированном окружении или в средах Linux (например, базовые контейнеры CI/CD).

* Для локальных тестов сетевой логики убедитесь, что вы запустили терминал от имени Администратора.
* Модульные тесты парсинга пакетов выносите в чистые функции, которые принимают сырые байты пакета (`&[u8]`) в качестве аргументов. Это позволяет тестировать логику разбора заголовков без реального перехвата трафика через WinDivert.
