# Воркфлоу актуализации документации G-Lock (/docs)

Этот навык запускает автоматический процесс проверки и обновления всей документации проекта G-Lock в директории `docs/` на основе последних изменений в исходном коде.

---

## 💡 Алгоритм выполнения воркфлоу

Когда пользователь вызывает команду `/docs`, вы должны выполнить следующие шаги:

### Шаг 1: Анализ последних изменений в коде
- Проверьте измененные файлы в Git (например, с помощью `git status`, `git diff` или просмотра истории недавних изменений).
- Определите, какие новые функции, виджеты, классы или правила фильтрации трафика были добавлены или модифицированы (например, переименование исполняемого файла, изменение настроек ограничителя скорости IPS, добавление сохранения ширины колонок лога и т.д.).

### Шаг 2: Обновление существующей документации
- Найдите файлы в папке `docs/`, которые описывают измененные модули, и внесите в них актуальные технические сведения.
- Обязательно обновите пользовательские инструкции:
  - [Инструкция_RU.txt](file:///g:/guardian-3.5.0/docs/Инструкция_RU.txt)
  - [Instructions_EN.txt](file:///g:/guardian-3.5.0/docs/Instructions_EN.txt)

### Шаг 3: Создание документации для новых функций
- Если в кодовую базу была добавлена крупная функциональность, которая ранее не была задокументирована (например, новая система синхронизации списков, новый виджет и т.д.), создайте для нее новый `.md` файл в папке `docs/` с подробным описанием архитектуры и логики работы.

### Шаг 4: Архивирование устаревших документов
- Если какие-то документы описывают полностью вырезанные или переписанные модули (например, старые отдельные классы фильтров `WhitelistSession` и `LockedSession`, которые были объединены в `PrivateSession`), перенесите эти файлы в архивную папку [docs/archive/](file:///g:/guardian-3.5.0/docs/archive).
- Убедитесь, что все ссылки в оставшихся файлах документации обновлены и не ведут на удаленные или перемещенные документы.

---

## 🎯 Результат выполнения
После завершения воркфлоу выведите пользователю краткий отчет:
1. Какие документы были обновлены.
2. Какие новые документы были созданы.
3. Какие файлы были перемещены в архив `docs/archive/`.
4. Список измененных ссылок (если применимо).
