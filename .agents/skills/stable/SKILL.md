# Воркфлоу автоматизации стабильного релиза G-Lock (/stable)

Этот навык автоматизирует выпуск новой стабильной версии приложения G-Lock, включая инкремент версии по стандарту SemVer, локальное тестирование/сборку, коммит версионных файлов в Git и создание/отправку тега версии на GitHub для запуска автосборки.

---

## 💡 Алгоритм выполнения воркфлоу

Когда пользователь вызывает команду `/stable`, вы должны выполнить следующие шаги:

### Шаг 1: Определение типа инкремента (SemVer)
- Проверьте, указал ли пользователь тип изменения версии: `major`, `minor` или `patch`.
- Если тип не указан явно, спросите пользователя или по умолчанию используйте **`patch`** (для багфиксов и рефакторинга).

### Шаг 2: Инкремент версии и сборка исполняемого файла
- Запустите команду сборщика с выбранным типом инкремента:
  ```powershell
  poetry run build [major|minor|patch]
  ```
  *(Например, `poetry run build patch`)*
- Убедитесь, что команда завершилась успешно (`exit code 0`). Если возникли ошибки компиляции, остановите воркфлоу и сообщите о них пользователю.

### Шаг 3: Чтение новой версии
- Прочитайте обновленную версию из файла `g_lock/__version__.py` (переменная `__version__`).

### Шаг 4: Коммит и отправка изменений в Git
- Добавьте в индекс (stage) измененные файлы версии:
  - `g_lock/__version__.py`
  - `spec/version.txt`
  - `poetry.lock` (если изменился)
- Закоммитьте изменения с сообщением:
  ```bash
  git commit -m "release: bump version to vX.Y.Z"
  ```
- Сделайте `git push`, чтобы отправить коммит в удаленную ветку `main`. Если пуш отклонен (например, нужно стянуть изменения), выполните `git pull --rebase` и повторите пуш.

### Шаг 5: Создание и отправка тега версии
- Создайте локальный Git-тег с именем версии (с префиксом `v`):
  ```bash
  git tag vX.Y.Z
  ```
  *(Например: `git tag v1.0.4`)*
- Отправьте тег на GitHub:
  ```bash
  git push origin v1.0.4
  ```

---

## 🎯 Результат выполнения

После успешного выполнения всех шагов выведите пользователю отчет:
1. Текущая установленная версия приложения (например, `1.0.4`).
2. Подтверждение успешной отправки коммита и тега на GitHub.
3. Инструкция для пользователя со ссылкой на вкладку компиляции релиза на GitHub Actions.
