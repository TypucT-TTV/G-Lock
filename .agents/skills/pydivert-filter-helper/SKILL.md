# Навык работы с WinDivert (Rust crate `windivert`) в G-Lock v2

Этот навык предназначен для написания корректных правил фильтрации, перехвата и анализа UDP пакетов в GTA Online с использованием Rust.

## 📌 Базовый фильтр WinDivert

Для захвата P2P-трафика GTA Online используется следующий фильтр-запрос:
```rust
let filter = "udp.DstPort == 6672 and udp.PayloadLength > 0 and ip";
```
* **Порт 6672**: Стандартный порт Rockstar для обмена данными между игроками по протоколу P2P.
* **PayloadLength > 0**: Исключает пустые пакеты без полезной нагрузки.
* **ip**: Захватывает только IPv4 пакеты.

## 🛠️ Основные паттерны использования windivert в Rust

При работе с библиотекой `windivert` следуйте этим шаблонам:

### 1. Открытие WinDivert сессии
```rust
use windivert::prelude::*;

let w: WinDivert<windivert::layer::NetworkLayer> = match WinDivert::network(filter, 0, WinDivertFlags::default()) {
    Ok(handle) => handle,
    Err(e) => {
        eprintln!("Failed to open WinDivert handle: {:?}", e);
        return;
    }
};
```

### 2. Чтение, модификация и отправка пакетов
Каждый перехваченный пакет должен быть либо отправлен дальше (`w.send(&packet)`), либо проигнорирован (отброшен):
```rust
let mut buf = [0u8; 65535];
while !*stop_flag.read() {
    let packet = match w.recv(Some(&mut buf)) {
        Ok(p) => p,
        Err(_) => continue,
    };

    if should_allow_packet(&packet) {
        let _ = w.send(&packet);  // Пропустить дальше в сетевой стек Windows
    } else {
        // Пакет отбрасывается автоматически, если мы не вызвали w.send()
    }
}
```

## 🔍 Эвристика по размеру полезной нагрузки (Payload Size)

Тип пакета определяется по длине его полезной нагрузки:

* **Heartbeat (Пульс)**: Размеры `12`, `18`, `63` байт.
  * *Правило*: Должны пропускаться **всегда** для всех IP-адресов, включая транзитные релеи, даже если сессия заблокирована/заперта (`is_locked = true`). Если заблокировать их, игрок потеряет связь с серверами авторизации/сохранениями.
* **Matchmaking (Попытка подключения)**: Размеры `191`, `207`, `223`, `239` байт.
  * *Правило*: Блокировка этих размеров при `is_locked = true` предотвращает появление новых игроков в сессии.

## ⚠️ Управление ресурсами и потоками

1. **Многопоточность**:
   - Чтение пакетов через `w.recv()` является блокирующей операцией.
   - Запускайте сетевой цикл в фоновом потоке: `std::thread::spawn(move || { ... })`.
   - Основной поток (Tauri/Svelte) не должен блокироваться, чтобы интерфейс оставался отзывчивым.
2. **Синхронизация**:
   - Используйте флаги отмены (`STOP_FLAG` через `RwLock<bool>`) для завершения цикла в потоке.
   - Закрытие хэндла `w` освобождает драйвер автоматически при выходе из области видимости (Drop).
