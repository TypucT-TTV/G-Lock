# Правила разработки проекта G-Lock для ИИ-помощников

Этот документ содержит стандарты кодирования, архитектурные требования и ограничения проекта **G-Lock** (нативного фаервола для GTA Online на базе Tauri + Rust + Svelte). Все агенты и ИИ-помощники обязаны следовать этим правилам при анализе и модификации кодовой базы.

---

## 1. Стандарты кода и типизация

* **Стек технологий**: Rust (бэкенд) + Svelte/TypeScript (фронтенд) + Tauri v2 (интеграция).
* **Rust**:
  * Код должен быть чистым, следовать идиоматическому Rust, проходить проверки `cargo clippy` и форматироваться с помощью `cargo fmt`.
  * Использовать потокобезопасные примитивы синхронизации (`RwLock` / `Mutex` из библиотеки `parking_lot`) для глобального состояния.
* **Frontend (Svelte 5 & TypeScript)**:
  * Использовать современный синтаксис Svelte 5 с рунами (`$state`, `$derived`, `$effect`).
  * Для обмена сообщениями с бэкендом использовать Tauri IPC (`invoke`, `listen`).
  * Стилизовать интерфейс с помощью Vanilla CSS в тегах `<style>`.

---

## 2. Разработка GUI (Tauri & Svelte)

* **Glassmorphism и премиальный дизайн**:
  * Интерфейс должен выглядеть современно и отзывчиво (полупрозрачные карточки, размытие заднего фона `backdrop-filter: blur()`, неоновые свечения, плавные анимации hover-эффектов).
* **Масштабирование и Zoom**:
  * Масштаб интерфейса (`Zoom`) изменяется по сочетаниям клавиш `Ctrl +`, `Ctrl -` и сбрасывается по `Ctrl 0`.
  * Масштабирование должно выполняться через CSS-трансформацию `transform: scale(zoomFactor)` на основном контейнере `.app-layout`.
  * Чтобы разметка сохраняла адаптивность без черных полей по бокам, ширина и высота контейнера должны рассчитываться как `100 / zoomFactor` в единицах `vw` и `vh` соответственно.
* **Сохранение состояния окна**:
  * Координаты (`window_x`, `window_y`) и внешние размеры окна (`window_width`, `window_height`) должны сохраняться в файл конфигурации `data.json`.
  * Для надежности размеры окна сохраняются с помощью метода `window.outer_size()` при событии `Resized`, а положение — при событии `Moved`. Данные сохраняются на диск при событии `CloseRequested`.
  * При сохранении настроек из Svelte бэкенд обязан сохранять текущие размеры, координаты окна и коэффициент зума в `STATE.config`, предотвращая их затирание пустыми значениями.

## 3. Сетевой стек и многопоточность

* **Работа с WinDivert (Rust)**:
  * Фильтрация входящего и исходящего трафика осуществляется через драйвер WinDivert с помощью Rust-библиотеки `windivert`.
  * Базовый фильтр для GTA Online: `(udp.SrcPort == 6672 or udp.DstPort == 6672) and udp.PayloadLength > 0 and ip` (только IPv4, P2P-порт GTA).
  * Peer IP определяется по направлению пакета: source для inbound и destination для outbound.
* **Многопоточная модель**:
  * Блокирующий захват и фильтрация пакетов (метод `w.recv()`) должны выполняться в отдельном потоке `std::thread::spawn`.
  * Основной поток Tauri (управляющий окном и веб-вью) не должен блокироваться сетевыми операциями.
  * Для обмена информацией между сетевым потоком и GUI используются Tauri-события: `app.emit("connection-log", ...)` для логов соединений и `app.emit("play-sound", ...)` для звукового сопровождения.
* **Восстановление драйвера**:
  * При Windows error 1058 разрешён один повторный запуск после `sc.exe config WinDivert start= demand` с путём к комплектному `WinDivert64.sys`. Приложение не должно останавливать или удалять службу.

## 4. Локализация (i18n)

* **Поддержка языков**: Приложение поддерживает русский и английский языки.
* **Запрет хардкода строк**:
  * Запрещено писать переводы текстов интерфейса напрямую в HTML-разметке.
  * Все строки должны извлекаться из словаря локализации `t` в `+page.svelte` с использованием реактивного объекта `activeLang`.
  * Новые строки необходимо добавлять в оба языковых блока `t.ru` и `t.en` в `+page.svelte`.

## 5. Сборка и версионирование

* **Имена артефактов**:
  * Cargo-бинарник называется `g-lock-tauri.exe`, а установщики Tauri — `G-Lock_<версия>_x64-setup.exe` и `G-Lock_<версия>_x64_en-US.msi`.
* **Семантическое версионирование (SemVer)**:
  * Номер версии имеет формат `MAJOR.MINOR.PATCH` (например, `2.0.8`).
  * **Обязательное обновление версии**:
    * **При каждом исправлении, доработке, изменении или багфиксе ИИ-помощник обязан обновить (инкрементировать PATCH-версию, например, с 2.0.7 на 2.0.8) версию приложения** во всех конфигурационных файлах проекта (`Cargo.toml`, `package.json`, `tauri.conf.json`) и в заголовках/интерфейсе GUI.
* **Автоматизация**:
  * Сборка релиза запускается командой `npm run tauri build`.
  * Итоговые установщики собираются в директории `src-tauri/target/release/bundle/nsis/` (exe-установщик) и `bundle/msi/` (msi-пакет).
  * **Ограничение автосборок в GitHub Actions**:
    * Запрещено собирать релизные инсталляторы в GitHub Actions на каждый обычный коммит или пуш в ветку `main`.
    * Сборка стабильного релиза на GitHub запускается исключительно по явной команде пользователя `/stable` (что приводит к созданию и пушу Git-тега версии `v*` или при ручном запуске `workflow_dispatch`).

## 6. Сетевая безопасность и исключение блокировок Rockstar

* **Heartbeat и приоритет blacklist**:
  * Служебные пульс-пакеты (heartbeats) с размерами payload `12`, `18` и `63` байт должны проходить для всех адресов, **кроме явно добавленных пользователем в постоянный blacklist**, включая транзитные релей-серверы Rockstar/Azure (`STATE.dynamic_blacklist`), даже если сессия заперта.
  * Постоянный blacklist имеет абсолютный приоритет над heartbeat, whitelist и кэшем известных peer; блокировка действует в обоих направлениях.
* **Режим Whitelist**:
  * В открытом состоянии разрешается служебный relay/matchmaking handshake без добавления relay IP в кэш, а прямой P2P-трафик проходит только для whitelist.
  * В запертом состоянии сохраняются уже известные peer и блокируются все новые адреса, включая новые whitelist IP. F9 переключает Open/Locked, не сбрасывая режим Whitelist.
* **Исключение из IPS (Rate Limiter)**:
  * Все IP-адреса, входящие в динамический черный список (релеи Rockstar/Azure, загружаемые из `db.json`), должны быть **полностью исключены из системы ограничения частоты запросов (IPS)**.
  * Файл базы данных IP-адресов `db.json` должен распространяться как Tauri-ресурс и читаться с использованием надежного разрешения путей в `load_dynamic_blacklist(app)`.
