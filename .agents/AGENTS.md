# Правила разработки проекта G-Lock для ИИ-помощников

Этот документ содержит стандарты кодирования, архитектурные требования и ограничения проекта **G-Lock** (кастомного фаервола для GTA Online). Все агенты и ИИ-помощники обязаны следовать этим правилам при анализе и модификации кодовой базы.

---

## 1. Стандарты кода и типизация

* **Версия Python**: Код должен быть совместим с Python версии `3.10` и выше.
* **Строгая типизация**: Весь новый код должен иметь полные аннотации типов (включая возвращаемые типы функций и методов).
  * Обязательно использование `from __future__ import annotations` в начале каждого файла.
  * Код должен успешно проходить проверку `mypy` в строгом режиме (`strict = true`), настроенном в [pyproject.toml](file:///g:/g_lock-3.5.0/pyproject.toml).
* **Форматирование кода**: Соблюдать форматирование с помощью утилит `black` и `isort`.
  * Длина строки ограничена 88 символами (`max-line-length = 88`).
  * Для сортировки импортов использовать профиль `black` (см. `[tool.isort]` в [pyproject.toml](file:///g:/g_lock-3.5.0/pyproject.toml)).

## 2. Разработка GUI (Tkinter & Canvas)

Интерфейс G-Lock построен на базе встроенной библиотеки `tkinter`, но имеет уникальную кастомную архитектуру для создания визуальных эффектов «стекла» (glassmorphism) и сглаживания:

* **Использование Canvas вместо виджетов**:
  * Большинство элементов на главном окне ([main_window.py](file:///g:/g_lock-3.5.0/g_lock/gui/main_window.py)) рисуются методами `create_*` непосредственно на одном общем холсте `tk.Canvas`.
  * Не используйте стандартные дочерние виджеты (вроде `tk.Button` или `ttk.Button`) поверх холста, так как Windows/Tkinter не поддерживают альфа-прозрачность для перекрывающихся виджетов (дочерний виджет будет иметь сплошной непрозрачный фон).
* **Кастомные кнопки**:
  * Для кнопок на основном Canvas используйте класс `CanvasButton` из [widgets.py](file:///g:/g_lock-3.5.0/g_lock/gui/widgets.py). Он рисует форму кнопки прямо на общем холсте и использует штриховку `stipple="gray50"` для смешивания цвета кнопки с фоновым рисунком, создавая эффект полупрозрачности.
  * Для диалоговых окон, где нет сложного фонового рисунка, допускается использовать автономный виджет `NeonButton` (также из [widgets.py](file:///g:/g_lock-3.5.0/g_lock/gui/widgets.py)).
* **Масштабирование DPI (DPI-Awareness)**:
  * Не хардкодьте геометрические размеры элементов в пикселях.
  * Все базовые размеры (например, `_BUTTON_WIDTH_BASE`) должны быть умножены на текущий коэффициент масштабирования экрана с помощью функции `dpi.get_scale_factor()` из [dpi.py](file:///g:/g_lock-3.5.0/g_lock/gui/dpi.py).
* **Сглаживание скруглений**:
  * Поскольку у `tkinter` нет встроенных примитивов для скругленных прямоугольников, используйте вспомогательную функцию `rounded_rect_points` из [theme.py](file:///g:/g_lock-3.5.0/g_lock/gui/theme.py) для генерации 12-точечного полигона и рисуйте его с параметром `smooth=True`.

## 3. Сетевой стек и многопроцессность

* **Работа с WinDivert**:
  * Фильтрация входящего трафика осуществляется через драйвер WinDivert с помощью библиотеки `pydivert`.
  * Базовый фильтр для GTA Online: `udp.DstPort == 6672 and udp.PayloadLength > 0 and ip` (только IPv4, P2P порт GTA).
* **Многопроцессная модель**:
  * Любые блокирующие операции захвата и фильтрации пакетов (методы `pydivert.WinDivert.recv()`) должны выполняться в отдельных процессах `multiprocessing.Process`.
  * **Категорически запрещено** выполнять блокирующий захват пакетов в основном потоке GUI или в стандартных потоках `threading.Thread` (это блокирует GIL и вешает интерфейс).
  * Для обмена информацией между фоновыми процессами фильтрации и GUI используйте потокобезопасные очереди `queue.Queue` и проксирование событий через `Context` ([context.py](file:///g:/g_lock-3.5.0/g_lock/dispatcher/context.py)).

## 4. Локализация (i18n)

* **Поддержка языков**: Приложение поддерживает русский и английский языки.
* **Запрет хардкода строк**:
  * Запрещено писать русские или английские тексты интерфейса напрямую в коде виджетов.
  * Все строки должны извлекаться с помощью функции `t("key")` из модуля локализации [i18n.py](file:///g:/g_lock-3.5.0/g_lock/gui/i18n.py).
  * Новые строки необходимо добавлять в словари перевода в [i18n.py](file:///g:/g_lock-3.5.0/g_lock/gui/i18n.py).

## 5. Сборка и версионирование

* **Имя исполняемого файла**:
  * Скомпилированный исполняемый файл приложения в папке `dist` должен всегда называться `G-Lock.exe` без указания версии.
  * Версия приложения должна отображаться в заголовке окна GUI (например, `G-Lock 1.0.0 - Открыто`) и в метаданных/свойствах самого `.exe` файла.
* **Семантическое версионирование (SemVer)**:
  * Номер версии имеет формат `MAJOR.MINOR.PATCH` (например, `1.0.0`).
  * Для изменения версии при сборке используются следующие параметры:
    * `poetry run build major` — крупное архитектурное обновление (`1.0.0` -> `2.0.0`).
    * `poetry run build minor` — добавление функциональности без нарушения совместимости (`1.0.0` -> `1.1.0`).
    * `poetry run build patch` (или без параметров) — багфиксы, исправления ошибок, рефакторинг (`1.0.0` -> `1.0.1`).
* **Автоматизация**:
  * Скрипт сборки `build.py` считывает текущую версию из файла `g_lock/__version__.py`, инкрементирует нужную часть версии, записывает ее обратно, обновляет файл `spec/version.txt` и запускает сборку PyInstaller.
  * Итоговый файл копируется/компилируется в `dist/G-Lock.exe`.

## 6. Сетевая безопасность и исключение блокировок Rockstar

* **Всегда разрешать Heartbeats**:
  * Служебные пульс-пакеты (heartbeats) с размерами payload `12`, `18` и `63` байт должны **всегда беспрепятственно проходить** для всех IP-адресов, включая транзитные релей-серверы Rockstar/Azure (`Menu.dynamic_blacklist`), даже если сессия заперта (`locked = True`).
  * Блокировка этих пакетов ломает соединение с облачными сохранениями и авторизацией Social Club, что приводит к бану внешнего IP пользователя на стороне серверов Rockstar.
* **Исключение из IPS (Rate Limiter)**:
  * Все IP-адреса, входящие в динамический черный список `Menu.dynamic_blacklist` (релеи Rockstar/Azure), должны быть **полностью исключены из системы ограничения частоты запросов (IPS)**.
  * Во время запуска игры и синхронизации с облаком трафик к этим серверам может кратковременно превышать пороги (например, 100–120 PPS), что при отсутствии исключения приводило к ложным банам серверов авторизации на стороне клиента и последующей потере связи.
